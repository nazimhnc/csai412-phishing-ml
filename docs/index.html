✅ Dataset Balance — Near-Perfect Ratio

  • Phishing — 4,898 samples (44.3%)
  • Legitimate — 6,157 samples (55.7%)

Near-perfect balance: 4,898 Phishing (44.3%) vs 6,157 Legitimate (55.7%). This 1.26:1 ratio means models cannot cheat by predicting the majority class — they must genuinely learn the patterns that distinguish phishing from legitimate websites.

Feature Space Characteristics: This dataset encodes 30 URL and website characteristics as categorical features ({-1, 0, 1}) representing suspicious, neutral, and legitimate states. Unlike continuous datasets, these discrete features create axis-aligned decision boundaries that tree-based models can exploit naturally, while kernel methods must work harder to find non-linear separations in this discrete feature space.

Key PCA Findings

  • With 30 discrete features ({-1, 0, 1}), PCA variance is more evenly distributed than with continuous data — no single component dominates, requiring more components to capture total variance.
  • The biplot reveals which URL/website features contribute most to the first two principal components — security-related features like SSLfinal_State, URL_of_Anchor, and having_Sub_Domain are expected to be major drivers.
  • The 2D scatter shows partial but not complete separation of phishing and legitimate classes, explaining why all supervised models achieve >92% but none reach 100% — some phishing sites mimic legitimate characteristics effectively.
  • 3D PCA provides better class separation, revealing that the phishing-legitimate boundary is a non-linear surface in the full 30-dimensional feature space.

This project applies multiple supervised and unsupervised machine learning algorithms to the UCI Phishing Websites Dataset (R.M. Mohammad et al., University of Huddersfield) to classify websites as phishing or legitimate based on URL and website characteristics.

Methodology

1
Data Collection & Preprocessing

Loaded UCI Phishing Websites dataset (11,055 samples, 30 categorical features encoded as {-1, 0, 1}). Applied StandardScaler normalization. No missing values. Stratified 80/20 train-test split preserving class proportions (44.3% phishing, 55.7% legitimate).

2
Exploratory Data Analysis (EDA)

Analyzed binary class distribution, feature correlations with target, per-class feature proportions, and URL characteristic distributions to understand phishing indicators and class separability.

3
Model Training & Hyperparameter Tuning

Trained 6 supervised models (KNN, SVM-RBF, SVM-Linear, Decision Tree, Logistic Regression, MLP) and 1 unsupervised model (K-Means). Used GridSearchCV with 5-fold stratified cross-validation for hyperparameter optimization.

4
Evaluation & Comparison

Evaluated all models using accuracy, weighted precision, weighted recall, and weighted F1 score. Generated confusion matrices, ROC curves, decision boundaries, and feature importance plots for each model.

5
Dimensionality Reduction & Clustering

Applied PCA for visualization and understanding feature contributions in the 30-dimensional discrete space. Performed K-Means clustering with elbow method and silhouette analysis to discover natural groupings among phishing and legitimate websites.

6
Dashboard & Deployment

Built an interactive Streamlit dashboard for exploration and prediction. Created a static web dashboard deployed to Cloudflare Pages for universal access without requiring Python.

Key Results Summary

  • Best Model: Multi-Layer Perceptron — 97.24% accuracy, 97.24% F1
  • Runner-up: Decision Tree — 97.11% accuracy, near-identical to MLP thanks to discrete features
  • Key Insight: Discrete categorical features ({-1, 0, 1}) strongly favor tree-based and neural network models over distance/kernel-based methods
  • Strength: Near-perfect 1.26:1 class balance ensures accuracy is meaningful — no majority-class inflation
  • PCA Finding: Variance is distributed across many components due to 30 discrete features; phishing/legitimate classes show partial but not complete separation in 2D
  • Security Implication: 97.24% accuracy means ~3% of phishing sites evade detection — these are sophisticated sites that successfully mimic legitimate feature patterns
